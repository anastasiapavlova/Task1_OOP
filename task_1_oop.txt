Построить иерархию классов для вывода графических фигур:  отрезок, прямоугольник, эллипс и т.д - не менее 6 фигур. 
Распределить классы по модулям. 
Создать список фигур в виде отдельного класса. 
В главном модуле программы добавить в список различные фигуры (статическая инициализация), после чего запустить рисование списка фигур. 
Рисовать фигуры не обязательно, допускается вывод фигур на экран в виде текстовых строк вида "Rectangle(10, 20, 30, 40) и цвет, т.е. просто консольного приложения будет достаточно.

Требования: 
1.	Правильное построение иерархии классов
2.	Придерживаться принципа SRP и OC.
3.	слабосвязанный код при создании списока фигур в виде отдельного класса. 
выбор типа с помощью оператора case/switch и множественного if делать нельзя).
4.	BLL не должен быть связан с PL - ом.
5.	учесть расширяемость кода и например еще и возможность добавления функциональности в закрытую иерерхию (Visitor)
6.	Проект должен разрабатываться с использование системы контроля версий git (выложить проект на github)
7.	Все методы, большие блоки кода должны содержать комментарии. Язык комментариев - английский.
